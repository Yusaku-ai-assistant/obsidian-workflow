# Daily Summary スキル

## Vaultのフォルダ構造

```
Vault/
 ├ 00_Inbox/          ← Web Clipperの吐き出し先（常にフラット）
 ├ 01_Clips/          ← 整理済みアーカイブ（YYYYMMDD/単位）
 ├ 02_DailySummary/   ← 日次まとめ（1ファイル/日）
 └ 99_Logs/           ← 処理ログ（1ファイル/日）
```

## フォルダの状態定義

| フォルダ | ファイルがある意味 |
|---|---|
| 00_Inbox | 未処理（要対応） |
| 01_Clips | 整理済み・アーカイブ済み |
| 02_DailySummary | まとめ生成済み |
| 99_Logs | 処理記録 |

## 基本原則

- **まとめ生成を先に、フォルダ整理を後に行う**（整理が失敗してもまとめは残す）
- 外部APIは使用しない
- MDファイルのみを対象とする

## Web Clipperが生成するfrontmatter形式

```yaml
---
title: 記事タイトル
source: https://example.com
date: YYYY-MM-DD
tags: []
---
```

frontmatterがない場合はファイル名・本文から情報を補完すること。

## タグ体系（DailySummary生成時に各記事へ付与すること）

各クリップMDファイルの `tags:` に、以下のタグ体系から適切なものを選んで書き込む。
Obsidianの階層タグ形式（`ジャンル/サブカテゴリ`）を使用する。

### ジャンル（何の分野か・複数可）

| タグ | 用途 |
|---|---|
| `ジャンル/AI` | AI・機械学習・LLM全般 |
| `ジャンル/テック` | IT技術・ソフトウェア・ハードウェア |
| `ジャンル/ビジネス` | 経営・マーケティング・業界動向 |
| `ジャンル/セキュリティ` | 情報セキュリティ・サイバー攻撃 |
| `ジャンル/ネットワーク` | インフラ・通信・クラウド |
| `ジャンル/ツール` | アプリ・サービス・プロダクト紹介 |

### 重要度（1つだけ）

| タグ | 用途 |
|---|---|
| `重要度/高` | 業務への影響大・すぐに対応が必要 |
| `重要度/中` | 近いうちに活用できそう |
| `重要度/低` | トレンド把握・ゆるく参考に |

### ソース（情報元の種類・1つだけ）

| タグ | 用途 |
|---|---|
| `ソース/公式` | 公式ブログ・リリースノート・ドキュメント |
| `ソース/ブログ` | 個人ブログ・Qiita・Zenn・note |
| `ソース/ニュース` | ニュースメディア・ITmedia・TechCrunch等 |
| `ソース/SNS` | Reddit・X・Hacker News等 |

### タグ付けのルール

- ジャンルは複数付けてよい（例：AIとツール両方該当なら両方付与）
- 重要度・ソースは必ず1つだけ選ぶ
- 迷ったら `重要度/低` を選ぶ
- `重要度/高` は業務直結の場合のみにとどめる

### タグ付けの例

```yaml
tags:
  - ジャンル/AI
  - ジャンル/ツール
  - 重要度/中
  - ソース/SNS
```
